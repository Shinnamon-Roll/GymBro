\section{คำอธิบาย ER Diagram}
แผนภาพ ER Diagram ข้างต้นแสดงความสัมพันธ์แบบ One-to-Many (1:N) ระหว่างตารางหลักต่างๆ ดังนี้:

\begin{enumerate}
    \item \textbf{Customers (1) $\rightarrow$ TrainingSessions (N)}: 
    ลูกค้าหนึ่งคนสามารถจองเซสชันการฝึกซ้อมได้หลายครั้ง แต่ละเซสชันจะเป็นของลูกค้าเพียงคนเดียว
    
    \item \textbf{Trainers (1) $\rightarrow$ TrainingSessions (N)}: 
    เทรนเนอร์หนึ่งคนสามารถดูแลการฝึกซ้อมได้หลายเซสชัน (ในช่วงเวลาที่ต่างกัน) แต่ละเซสชันจะมีเทรนเนอร์ผู้ดูแลเพียงคนเดียว
    
    \item \textbf{GymEquipments (1) $\rightarrow$ TrainingSessions (N)}: 
    อุปกรณ์หนึ่งชิ้นสามารถถูกใช้งานได้ในหลายเซสชัน (ในช่วงเวลาที่ต่างกัน) แต่ละเซสชันจะมีการจองใช้อุปกรณ์หลักหนึ่งชิ้น
\end{enumerate}

\section{รายละเอียดตาราง (Table Schema)}

\subsection{ตาราง Customers (ข้อมูลสมาชิก)}
เก็บข้อมูลส่วนตัวของสมาชิกที่มาใช้บริการยิม
\begin{itemize}
    \item \textbf{id} (PK): รหัสสมาชิก (Integer, Auto Increment)
    \item \textbf{fullName}: ชื่อ-นามสกุล (String 150)
    \item \textbf{email}: อีเมล (String 150, Unique) ใช้สำหรับเข้าสู่ระบบ
    \item \textbf{phone}: เบอร์โทรศัพท์ (String 20)
    \item \textbf{memberType}: ประเภทสมาชิก (Standard, Premium, VIP)
    \item \textbf{memberLevel}: ระดับสมาชิก (Basic, Silver, Gold)
    \item \textbf{memberStartDate}: วันที่เริ่มเป็นสมาชิก
    \item \textbf{memberEndDate}: วันที่สิ้นสุดสมาชิก
\end{itemize}

\subsection{ตาราง Trainers (ข้อมูลเทรนเนอร์)}
เก็บข้อมูลรายชื่อและรายละเอียดของเทรนเนอร์
\begin{itemize}
    \item \textbf{id} (PK): รหัสเทรนเนอร์ (Integer, Auto Increment)
    \item \textbf{trainerName}: ชื่อเทรนเนอร์ (String 150)
    \item \textbf{trainerLevel}: ระดับของเทรนเนอร์ (Junior, Senior, Master)
    \item \textbf{specialty}: ความเชี่ยวชาญพิเศษ (String 100)
    \item \textbf{phone}: เบอร์โทรศัพท์ติดต่อ (String 20)
\end{itemize}

\subsection{ตาราง GymEquipments (ข้อมูลอุปกรณ์)}
เก็บข้อมูลอุปกรณ์ออกกำลังกายที่มีให้บริการ
\begin{itemize}
    \item \textbf{id} (PK): รหัสอุปกรณ์ (Integer, Auto Increment)
    \item \textbf{equipmentName}: ชื่ออุปกรณ์ (String 150)
    \item \textbf{category}: หมวดหมู่อุปกรณ์ (Cardio, Strength, etc.)
    \item \textbf{status}: สถานะปัจจุบัน (Available, In Use, Maintenance)
\end{itemize}

\subsection{ตาราง TrainingSessions (ตารางการฝึกซ้อม)}
เก็บข้อมูลการจองคลาสหรือเซสชันการฝึกซ้อม เชื่อมโยงลูกค้า เทรนเนอร์ และอุปกรณ์เข้าด้วยกัน
\begin{itemize}
    \item \textbf{id} (PK): รหัสการจอง (Integer, Auto Increment)
    \item \textbf{sessionDate}: วันและเวลาเริ่มต้นการฝึกซ้อม (DateTime)
    \item \textbf{endDate}: วันและเวลาสิ้นสุดการฝึกซ้อม (DateTime)
    \item \textbf{customerId} (FK): อ้างอิงไปยังตาราง Customers
    \item \textbf{trainerId} (FK): อ้างอิงไปยังตาราง Trainers
    \item \textbf{equipmentId} (FK): อ้างอิงไปยังตาราง GymEquipments
\end{itemize}
